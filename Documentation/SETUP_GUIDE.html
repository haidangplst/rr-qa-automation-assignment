🎯 QA Automation Framework Setup

Quick Start

Windows: Double-click setup.bat

Mac/Linux: Run bash setup-tests.sh

Prerequisites

  • .NET 10.0 SDK or later
  • Visual Studio 2022 or Visual Studio Code
  • Chrome browser (for Chrome-based testing)

Manual Setup

Step 1: Install Playwright Browsers

dotnet tool install --global Microsoft.Playwright.CLI
playwright install chromium

Step 2: Restore Dependencies

dotnet restore

Step 3: Build Project

dotnet build

Running Tests

Run All Tests

dotnet test

Run Specific Test Class

dotnet test --filter "ClassName=ExamplePageTests"

Run with Verbose Output

dotnet test --verbosity detailed

Run Tests in Headed Mode (See Browser)

Edit PlaywrightTests/Fixtures/BrowserFixture.cs and change:

Headless = true // Change to false

Project Structure

  • PageObjects/ - Page Object Model classes
  • Fixtures/ - Test fixtures for browser setup/teardown
  • Tests/ - Test classes using POM pattern
  • Utilities/ - Helper utilities for common operations

Creating New Page Objects

See FRAMEWORK_GUIDE.md for detailed instructions on creating page objects and tests.

Chrome Configuration

Chrome is configured with the following options:

  • Headless mode (disabled automation detection)
  • Memory optimization for CI/CD
  • HTTPS errors ignored for testing

Configure in: PlaywrightTests/PlaywrightFixture.cs

Documentation

  • FRAMEWORK_GUIDE.md - Comprehensive framework documentation
  • PlaywrightTests/PageObjects/ExamplePage.cs - Example page object
  • PlaywrightTests/Tests/ExamplePageTests.cs - Example tests using POM

Troubleshooting

Playwright browsers not found: Run playwright install chromium

Port already in use: Change the port configuration in browser launch options

Tests timing out: Increase timeout values in BasePage methods
